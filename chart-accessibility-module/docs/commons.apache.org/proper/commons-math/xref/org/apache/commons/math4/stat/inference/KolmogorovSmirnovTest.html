50  /**
51   * Implementation of the <a href="http://en.wikipedia.org/wiki/Kolmogorov-Smirnov_test">
52   * Kolmogorov-Smirnov (K-S) test</a> for equality of continuous distributions.
53   * <p>
54   * The K-S test uses a statistic based on the maximum deviation of the empirical distribution of
55   * sample data points from the distribution expected under the null hypothesis. For one-sample tests
56   * evaluating the null hypothesis that a set of sample data points follow a given distribution, the
57   * test statistic is \(D_n=\sup_x |F_n(x)-F(x)|\), where \(F\) is the expected distribution and
58   * \(F_n\) is the empirical distribution of the \(n\) sample data points. The distribution of
59   * \(D_n\) is estimated using a method based on [1] with certain quick decisions for extreme values
60   * given in [2].
61   * </p>
62   * <p>
63   * Two-sample tests are also supported, evaluating the null hypothesis that the two samples
64   * {@code x} and {@code y} come from the same underlying distribution. In this case, the test
65   * statistic is \(D_{n,m}=\sup_t | F_n(t)-F_m(t)|\) where \(n\) is the length of {@code x}, \(m\) is
66   * the length of {@code y}, \(F_n\) is the empirical distribution that puts mass \(1/n\) at each of
67   * the values in {@code x} and \(F_m\) is the empirical distribution of the {@code y} values. The
68   * default 2-sample test method, {@link #kolmogorovSmirnovTest(double[], double[])} works as
69   * follows:
70   * <ul>
71   * <li>For small samples (where the product of the sample sizes is less than
72   * {@value #LARGE_SAMPLE_PRODUCT}), the method presented in [4] is used to compute the
73   * exact p-value for the 2-sample test.</li>
74   * <li>When the product of the sample sizes exceeds {@value #LARGE_SAMPLE_PRODUCT}, the asymptotic
75   * distribution of \(D_{n,m}\) is used. See {@link #approximateP(double, int, int)} for details on
76   * the approximation.</li>
77   * </ul></p><p>
78   * If the product of the sample sizes is less than {@value #LARGE_SAMPLE_PRODUCT} and the sample
79   * data contains ties, random jitter is added to the sample data to break ties before applying
80   * the algorithm above. Alternatively, the {@link #bootstrap(double[], double[], int, boolean)}
81   * method, modeled after <a href="http://sekhon.berkeley.edu/matching/ks.boot.html">ks.boot</a>
82   * in the R Matching package [3], can be used if ties are known to be present in the data.
83   * </p>
84   * <p>
85   * In the two-sample case, \(D_{n,m}\) has a discrete distribution. This makes the p-value
86   * associated with the null hypothesis \(H_0 : D_{n,m} \ge d \) differ from \(H_0 : D_{n,m} > d \)
87   * by the mass of the observed value \(d\). To distinguish these, the two-sample tests use a boolean
88   * {@code strict} parameter. This parameter is ignored for large samples.
89   * </p>
90   * <p>
91   * The methods used by the 2-sample default implementation are also exposed directly:
92   * <ul>
93   * <li>{@link #exactP(double, int, int, boolean)} computes exact 2-sample p-values</li>
94   * <li>{@link #approximateP(double, int, int)} uses the asymptotic distribution The {@code boolean}
95   * arguments in the first two methods allow the probability used to estimate the p-value to be
96   * expressed using strict or non-strict inequality. See
97   * {@link #kolmogorovSmirnovTest(double[], double[], boolean)}.</li>
98   * </ul>
99   * </p>
100  * <p>
101  * References:
102  * <ul>
103  * <li>[1] <a href="http://www.jstatsoft.org/v08/i18/"> Evaluating Kolmogorov's Distribution</a> by
104  * George Marsaglia, Wai Wan Tsang, and Jingbo Wang</li>
105  * <li>[2] <a href="http://www.jstatsoft.org/v39/i11/"> Computing the Two-Sided Kolmogorov-Smirnov
106  * Distribution</a> by Richard Simard and Pierre L'Ecuyer</li>
107  * <li>[3] Jasjeet S. Sekhon. 2011. <a href="http://www.jstatsoft.org/article/view/v042i07">
108  * Multivariate and Propensity Score Matching Software with Automated Balance Optimization:
109  * The Matching package for R</a> Journal of Statistical Software, 42(7): 1-52.</li>
110  * <li>[4] Wilcox, Rand. 2012. Introduction to Robust Estimation and Hypothesis Testing,
111  * Chapter 5, 3rd Ed. Academic Press.</li>
112  * </ul>
113  * <br/>
114  * Note that [1] contains an error in computing h, refer to <a
115  * href="https://issues.apache.org/jira/browse/MATH-437">MATH-437</a> for details.
116  * </p>
117  *
118  * @since 3.3
119  */

191     /**
192      * Computes the one-sample Kolmogorov-Smirnov test statistic, \(D_n=\sup_x |F_n(x)-F(x)|\) where
193      * \(F\) is the distribution (cdf) function associated with {@code distribution}, \(n\) is the
194      * length of {@code data} and \(F_n\) is the empirical distribution that puts mass \(1/n\) at
195      * each of the values in {@code data}.
196      *
197      * @param distribution reference distribution
198      * @param data sample being evaluated
199      * @return Kolmogorov-Smirnov statistic \(D_n\)
200      * @throws InsufficientDataException if {@code data} does not have length at least 2
201      * @throws NullArgumentException if {@code data} is null
202      */
203     public double kolmogorovSmirnovStatistic(RealDistribution distribution, double[] data) {
204         checkArray(data);
205         final int n = data.length;
206         final double nd = n;
207         final double[] dataCopy = new double[n];
208         System.arraycopy(data, 0, dataCopy, 0, n);
209         Arrays.sort(dataCopy);
210         double d = 0d;
211         for (int i = 1; i <= n; i++) {
212             final double yi = distribution.cumulativeProbability(dataCopy[i - 1]);
213             final double currD = FastMath.max(yi - (i - 1) / nd, i / nd - yi);
214             if (currD > d) {
215                 d = currD;
216             }
217         }
218         return d;
219     }

221     /**
222      * Computes the <i>p-value</i>, or <i>observed significance level</i>, of a two-sample <a
223      * href="http://en.wikipedia.org/wiki/Kolmogorov-Smirnov_test"> Kolmogorov-Smirnov test</a>
224      * evaluating the null hypothesis that {@code x} and {@code y} are samples drawn from the same
225      * probability distribution. Specifically, what is returned is an estimate of the probability
226      * that the {@link #kolmogorovSmirnovStatistic(double[], double[])} associated with a randomly
227      * selected partition of the combined sample into subsamples of sizes {@code x.length} and
228      * {@code y.length} will strictly exceed (if {@code strict} is {@code true}) or be at least as
229      * large as {@code strict = false}) as {@code kolmogorovSmirnovStatistic(x, y)}.
230      * <ul>
231      * <li>For small samples (where the product of the sample sizes is less than
232      * {@value #LARGE_SAMPLE_PRODUCT}), the exact p-value is computed using the method presented
233      * in [4], implemented in {@link #exactP(double, int, int, boolean)}. </li>
234      * <li>When the product of the sample sizes exceeds {@value #LARGE_SAMPLE_PRODUCT}, the
235      * asymptotic distribution of \(D_{n,m}\) is used. See {@link #approximateP(double, int, int)}
236      * for details on the approximation.</li>
237      * </ul><p>
238      * If {@code x.length * y.length} < {@value #LARGE_SAMPLE_PRODUCT} and the combined set of values in
239      * {@code x} and {@code y} contains ties, random jitter is added to {@code x} and {@code y} to
240      * break ties before computing \(D_{n,m}\) and the p-value. The jitter is uniformly distributed
241      * on (-minDelta / 2, minDelta / 2) where minDelta is the smallest pairwise difference between
242      * values in the combined sample.</p>
243      * <p>
244      * If ties are known to be present in the data, {@link #bootstrap(double[], double[], int, boolean)}
245      * may be used as an alternative method for estimating the p-value.</p>
246      *
247      * @param x first sample dataset
248      * @param y second sample dataset
249      * @param strict whether or not the probability to compute is expressed as a strict inequality
250      *        (ignored for large samples)
251      * @return p-value associated with the null hypothesis that {@code x} and {@code y} represent
252      *         samples from the same distribution
253      * @throws InsufficientDataException if either {@code x} or {@code y} does not have length at
254      *         least 2
255      * @throws NullArgumentException if either {@code x} or {@code y} is null
256      * @see #bootstrap(double[], double[], int, boolean)
257      */
258     public double kolmogorovSmirnovTest(double[] x, double[] y, boolean strict) {
259         final long lengthProduct = (long) x.length * y.length;
260         double[] xa = null;
261         double[] ya = null;
262         if (lengthProduct < LARGE_SAMPLE_PRODUCT && hasTies(x,y)) {
263             xa = MathArrays.copyOf(x);
264             ya = MathArrays.copyOf(y);
265             fixTies(xa, ya);
266         } else {
267             xa = x;
268             ya = y;
269         }
270         if (lengthProduct < LARGE_SAMPLE_PRODUCT) {
271             return exactP(kolmogorovSmirnovStatistic(xa, ya), x.length, y.length, strict);
272         }
273         return approximateP(kolmogorovSmirnovStatistic(x, y), x.length, y.length);
274     }

313     /**
314      * Computes the two-sample Kolmogorov-Smirnov test statistic, \(D_{n,m}=\sup_x |F_n(x)-F_m(x)|\)
315      * where \(n\) is the length of {@code x}, \(m\) is the length of {@code y}, \(F_n\) is the
316      * empirical distribution that puts mass \(1/n\) at each of the values in {@code x} and \(F_m\)
317      * is the empirical distribution of the {@code y} values. Finally \(n m D_{n,m}\) is returned
318      * as long value.
319      *
320      * @param x first sample
321      * @param y second sample
322      * @return test statistic \(n m D_{n,m}\) used to evaluate the null hypothesis that {@code x} and
323      *         {@code y} represent samples from the same underlying distribution
324      * @throws InsufficientDataException if either {@code x} or {@code y} does not have length at
325      *         least 2
326      * @throws NullArgumentException if either {@code x} or {@code y} is null
327      */
328     private long integralKolmogorovSmirnovStatistic(double[] x, double[] y) {
329         checkArray(x);
330         checkArray(y);
331         // Copy and sort the sample arrays
332         final double[] sx = MathArrays.copyOf(x);
333         final double[] sy = MathArrays.copyOf(y);
334         Arrays.sort(sx);
335         Arrays.sort(sy);
336         final int n = sx.length;
337         final int m = sy.length;
338 
339         int rankX = 0;
340         int rankY = 0;
341         long curD = 0l;
342 
343         // Find the max difference between cdf_x and cdf_y
344         long supD = 0l;
345         do {
346             double z = Double.compare(sx[rankX], sy[rankY]) <= 0 ? sx[rankX] : sy[rankY];
347             while(rankX < n && Double.compare(sx[rankX], z) == 0) {
348                 rankX += 1;
349                 curD += m;
350             }
351             while(rankY < m && Double.compare(sy[rankY], z) == 0) {
352                 rankY += 1;
353                 curD -= n;
354             }
355             if (curD > supD) {
356                 supD = curD;
357             }
358             else if (-curD > supD) {
359                 supD = -curD;
360             }
361         } while(rankX < n && rankY < m);
362         return supD;
363     }

400     /**
401      * Estimates the <i>p-value</i> of a two-sample
402      * <a href="http://en.wikipedia.org/wiki/Kolmogorov-Smirnov_test"> Kolmogorov-Smirnov test</a>
403      * evaluating the null hypothesis that {@code x} and {@code y} are samples drawn from the same
404      * probability distribution. This method estimates the p-value by repeatedly sampling sets of size
405      * {@code x.length} and {@code y.length} from the empirical distribution of the combined sample.
406      * When {@code strict} is true, this is equivalent to the algorithm implemented in the R function
407      * {@code ks.boot}, described in <pre>
408      * Jasjeet S. Sekhon. 2011. 'Multivariate and Propensity Score Matching
409      * Software with Automated Balance Optimization: The Matching package for R.'
410      * Journal of Statistical Software, 42(7): 1-52.
411      * </pre>
412      * @param x first sample
413      * @param y second sample
414      * @param iterations number of bootstrap resampling iterations
415      * @param strict whether or not the null hypothesis is expressed as a strict inequality
416      * @return estimated p-value
417      */
418     public double bootstrap(double[] x, double[] y, int iterations, boolean strict) {
419         final int xLength = x.length;
420         final int yLength = y.length;
421         final double[] combined = new double[xLength + yLength];
422         System.arraycopy(x, 0, combined, 0, xLength);
423         System.arraycopy(y, 0, combined, xLength, yLength);
424         final RealDistribution.Sampler sampler = new EnumeratedRealDistribution(combined).createSampler(rng);
425         final long d = integralKolmogorovSmirnovStatistic(x, y);
426         int greaterCount = 0;
427         int equalCount = 0;
428         double[] curX;
429         double[] curY;
430         long curD;
431         for (int i = 0; i < iterations; i++) {
432             curX = AbstractRealDistribution.sample(xLength, sampler);
433             curY = AbstractRealDistribution.sample(yLength, sampler);
434             curD = integralKolmogorovSmirnovStatistic(curX, curY);
435             if (curD > d) {
436                 greaterCount++;
437             } else if (curD == d) {
438                 equalCount++;
439             }
440         }
441         return strict ? greaterCount / (double) iterations :
442             (greaterCount + equalCount) / (double) iterations;
443     }

496     /**
497      * Calculates {@code P(D_n < d)} using method described in [1] with quick decisions for extreme
498      * values given in [2] (see above).
499      *
500      * @param d statistic
501      * @param n sample size
502      * @param exact whether the probability should be calculated exact using
503      *        {@link org.apache.commons.math4.fraction.BigFraction} everywhere at the expense of
504      *        very slow execution time, or if {@code double} should be used convenient places to
505      *        gain speed. Almost never choose {@code true} in real applications unless you are very
506      *        sure; {@code true} is almost solely for verification purposes.
507      * @return \(P(D_n < d)\)
508      * @throws MathArithmeticException if algorithm fails to convert {@code h} to a
509      *         {@link org.apache.commons.math4.fraction.BigFraction} in expressing {@code d} as \((k
510      *         - h) / m\) for integer {@code k, m} and \(0 \le h < 1\).
511      */
512     public double cdf(double d, int n, boolean exact)
513         throws MathArithmeticException {
514 
515         final double ninv = 1 / ((double) n);
516         final double ninvhalf = 0.5 * ninv;
517 
518         if (d <= ninvhalf) {
519             return 0;
520         } else if (ninvhalf < d && d <= ninv) {
521             double res = 1;
522             final double f = 2 * d - ninv;
523             // n! f^n = n*f * (n-1)*f * ... * 1*x
524             for (int i = 1; i <= n; ++i) {
525                 res *= i * f;
526             }
527             return res;
528         } else if (1 - ninv <= d && d < 1) {
529             return 1 - 2 * Math.pow(1 - d, n);
530         } else if (1 <= d) {
531             return 1;
532         }
533         if (exact) {
534             return exactK(d, n);
535         }
536         if (n <= 140) {
537             return roundedK(d, n);
538         }
539         return pelzGood(d, n);
540     }

542     /**
543      * Calculates the exact value of {@code P(D_n < d)} using the method described in [1] (reference
544      * in class javadoc above) and {@link org.apache.commons.math4.fraction.BigFraction} (see
545      * above).
546      *
547      * @param d statistic
548      * @param n sample size
549      * @return the two-sided probability of \(P(D_n < d)\)
550      * @throws MathArithmeticException if algorithm fails to convert {@code h} to a
551      *         {@link org.apache.commons.math4.fraction.BigFraction} in expressing {@code d} as \((k
552      *         - h) / m\) for integer {@code k, m} and \(0 \le h < 1\).
553      */
554     private double exactK(double d, int n)
555         throws MathArithmeticException {
556 
557         final int k = (int) Math.ceil(n * d);
558 
559         final FieldMatrix<BigFraction> H = this.createExactH(d, n);
560         final FieldMatrix<BigFraction> Hpower = H.power(n);
561 
562         BigFraction pFrac = Hpower.getEntry(k - 1, k - 1);
563 
564         for (int i = 1; i <= n; ++i) {
565             pFrac = pFrac.multiply(i).divide(n);
566         }
567 
568         /*
569          * BigFraction.doubleValue converts numerator to double and the denominator to double and
570          * divides afterwards. That gives NaN quite easy. This does not (scale is the number of
571          * digits):
572          */
573         return pFrac.bigDecimalValue(20, BigDecimal.ROUND_HALF_UP).doubleValue();
574     }

578      *
579      * @param d statistic
580      * @param n sample size
581      * @return \(P(D_n < d)\)
582      */

584

588 
589         double pFrac = Hpower.getEntry(k - 1, k - 1);

597     /**
598      * Computes the Pelz-Good approximation for \(P(D_n < d)\) as described in [2] in the class javadoc.
599      *
600      * @param d value of d-statistic (x in [2])
601      * @param n sample size
602      * @return \(P(D_n < d)\)
603      * @since 3.4
604      */
605     public double pelzGood(double d, int n) {
606         // Change the variable since approximation is for the distribution evaluated at d / sqrt(n)
607         final double sqrtN = FastMath.sqrt(n);
608         final double z = d * sqrtN;
609         final double z2 = d * d * n;
610         final double z4 = z2 * z2;
611         final double z6 = z4 * z2;
612         final double z8 = z4 * z4;
613 
614         // Eventual return value
615         double ret = 0;
616 
617         // Compute K_0(z)
618         double sum = 0;
619         double increment = 0;
620         double kTerm = 0;
621         double z2Term = MathUtils.PI_SQUARED / (8 * z2);
622         int k = 1;
623         for (; k < MAXIMUM_PARTIAL_SUM_COUNT; k++) {
624             kTerm = 2 * k - 1;
625             increment = FastMath.exp(-z2Term * kTerm * kTerm);
626             sum += increment;
627             if (increment <= PG_SUM_RELATIVE_ERROR * sum) {
628                 break;
629             }
630         }
631         if (k == MAXIMUM_PARTIAL_SUM_COUNT) {
632             throw new TooManyIterationsException(MAXIMUM_PARTIAL_SUM_COUNT);
633         }
634         ret = sum * FastMath.sqrt(2 * FastMath.PI) / z;
635 
636         // K_1(z)
637         // Sum is -inf to inf, but k term is always (k + 1/2) ^ 2, so really have
638         // twice the sum from k = 0 to inf (k = -1 is same as 0, -2 same as 1, ...)
639         final double twoZ2 = 2 * z2;
640         sum = 0;
641         kTerm = 0;
642         double kTerm2 = 0;
643         for (k = 0; k < MAXIMUM_PARTIAL_SUM_COUNT; k++) {
644             kTerm = k + 0.5;
645             kTerm2 = kTerm * kTerm;
646             increment = (MathUtils.PI_SQUARED * kTerm2 - z2) * FastMath.exp(-MathUtils.PI_SQUARED * kTerm2 / twoZ2);
647             sum += increment;
648             if (FastMath.abs(increment) < PG_SUM_RELATIVE_ERROR * FastMath.abs(sum)) {
649                 break;
650             }
651         }
652         if (k == MAXIMUM_PARTIAL_SUM_COUNT) {
653             throw new TooManyIterationsException(MAXIMUM_PARTIAL_SUM_COUNT);
654         }
655         final double sqrtHalfPi = FastMath.sqrt(FastMath.PI / 2);
656         // Instead of doubling sum, divide by 3 instead of 6
657         ret += sum * sqrtHalfPi / (3 * z4 * sqrtN);
658 
659         // K_2(z)
660         // Same drill as K_1, but with two doubly infinite sums, all k terms are even powers.
661         final double z4Term = 2 * z4;
662         final double z6Term = 6 * z6;
663         z2Term = 5 * z2;
664         final double pi4 = MathUtils.PI_SQUARED * MathUtils.PI_SQUARED;
665         sum = 0;
666         kTerm = 0;
667         kTerm2 = 0;
668         for (k = 0; k < MAXIMUM_PARTIAL_SUM_COUNT; k++) {
669             kTerm = k + 0.5;
670             kTerm2 = kTerm * kTerm;
671             increment =  (z6Term + z4Term + MathUtils.PI_SQUARED * (z4Term - z2Term) * kTerm2 +
672                     pi4 * (1 - twoZ2) * kTerm2 * kTerm2) * FastMath.exp(-MathUtils.PI_SQUARED * kTerm2 / twoZ2);
673             sum += increment;
674             if (FastMath.abs(increment) < PG_SUM_RELATIVE_ERROR * FastMath.abs(sum)) {
675                 break;
676             }
677         }
678         if (k == MAXIMUM_PARTIAL_SUM_COUNT) {
679             throw new TooManyIterationsException(MAXIMUM_PARTIAL_SUM_COUNT);
680         }
681         double sum2 = 0;
682         kTerm2 = 0;
683         for (k = 1; k < MAXIMUM_PARTIAL_SUM_COUNT; k++) {
684             kTerm2 = k * k;
685             increment = MathUtils.PI_SQUARED * kTerm2 * FastMath.exp(-MathUtils.PI_SQUARED * kTerm2 / twoZ2);
686             sum2 += increment;
687             if (FastMath.abs(increment) < PG_SUM_RELATIVE_ERROR * FastMath.abs(sum2)) {
688                 break;
689             }
690         }
691         if (k == MAXIMUM_PARTIAL_SUM_COUNT) {
692             throw new TooManyIterationsException(MAXIMUM_PARTIAL_SUM_COUNT);
693         }
694         // Again, adjust coefficients instead of doubling sum, sum2
695         ret += (sqrtHalfPi / n) * (sum / (36 * z2 * z2 * z2 * z) - sum2 / (18 * z2 * z));
696 
697         // K_3(z) One more time with feeling - two doubly infinite sums, all k powers even.
698         // Multiply coefficient denominators by 2, so omit doubling sums.
699         final double pi6 = pi4 * MathUtils.PI_SQUARED;
700         sum = 0;
701         double kTerm4 = 0;
702         double kTerm6 = 0;
703         for (k = 0; k < MAXIMUM_PARTIAL_SUM_COUNT; k++) {
704             kTerm = k + 0.5;
705             kTerm2 = kTerm * kTerm;
706             kTerm4 = kTerm2 * kTerm2;
707             kTerm6 = kTerm4 * kTerm2;
708             increment = (pi6 * kTerm6 * (5 - 30 * z2) + pi4 * kTerm4 * (-60 * z2 + 212 * z4) +
709                             MathUtils.PI_SQUARED * kTerm2 * (135 * z4 - 96 * z6) - 30 * z6 - 90 * z8) *
710                     FastMath.exp(-MathUtils.PI_SQUARED * kTerm2 / twoZ2);
711             sum += increment;
712             if (FastMath.abs(increment) < PG_SUM_RELATIVE_ERROR * FastMath.abs(sum)) {
713                 break;
714             }
715         }
716         if (k == MAXIMUM_PARTIAL_SUM_COUNT) {
717             throw new TooManyIterationsException(MAXIMUM_PARTIAL_SUM_COUNT);
718         }
719         sum2 = 0;
720         for (k = 1; k < MAXIMUM_PARTIAL_SUM_COUNT; k++) {
721             kTerm2 = k * k;
722             kTerm4 = kTerm2 * kTerm2;
723             increment = (-pi4 * kTerm4 + 3 * MathUtils.PI_SQUARED * kTerm2 * z2) *
724                     FastMath.exp(-MathUtils.PI_SQUARED * kTerm2 / twoZ2);
725             sum2 += increment;
726             if (FastMath.abs(increment) < PG_SUM_RELATIVE_ERROR * FastMath.abs(sum2)) {
727                 break;
728             }
729         }
730         if (k == MAXIMUM_PARTIAL_SUM_COUNT) {
731             throw new TooManyIterationsException(MAXIMUM_PARTIAL_SUM_COUNT);
732         }
733         return ret + (sqrtHalfPi / (sqrtN * n)) * (sum / (3240 * z6 * z4) +
734                 + sum2 / (108 * z6));
735 
736     }

738     /***
739      * Creates {@code H} of size {@code m x m} as described in [1] (see above).
740      *
741      * @param d statistic
742      * @param n sample size
743      * @return H matrix
744      * @throws NumberIsTooLargeException if fractional part is greater than 1
745      * @throws FractionConversionException if algorithm fails to convert {@code h} to a
746      *         {@link org.apache.commons.math4.fraction.BigFraction} in expressing {@code d} as \((k
747      *         - h) / m\) for integer {@code k, m} and \(0 <= h < 1\).
748      */
749     private FieldMatrix<BigFraction> createExactH(double d, int n)
750         throws NumberIsTooLargeException, FractionConversionException {
751 
752         final int k = (int) Math.ceil(n * d);
753         final int m = 2 * k - 1;
754         final double hDouble = k - n * d;
755         if (hDouble >= 1) {
756             throw new NumberIsTooLargeException(hDouble, 1.0, false);
757         }
758         BigFraction h = null;
759         try {
760             h = new BigFraction(hDouble, 1.0e-20, 10000);
761         } catch (final FractionConversionException e1) {
762             try {
763                 h = new BigFraction(hDouble, 1.0e-10, 10000);
764             } catch (final FractionConversionException e2) {
765                 h = new BigFraction(hDouble, 1.0e-5, 10000);
766             }
767         }
768         final BigFraction[][] Hdata = new BigFraction[m][m];
769 
770         /*
771          * Start by filling everything with either 0 or 1.
772          */
773         for (int i = 0; i < m; ++i) {
774             for (int j = 0; j < m; ++j) {
775                 if (i - j + 1 < 0) {
776                     Hdata[i][j] = BigFraction.ZERO;
777                 } else {
778                     Hdata[i][j] = BigFraction.ONE;
779                 }
780             }
781         }
782 
783         /*
784          * Setting up power-array to avoid calculating the same value twice: hPowers[0] = h^1 ...
785          * hPowers[m-1] = h^m
786          */
787         final BigFraction[] hPowers = new BigFraction[m];
788         hPowers[0] = h;
789         for (int i = 1; i < m; ++i) {
790             hPowers[i] = h.multiply(hPowers[i - 1]);
791         }
792 
793         /*
794          * First column and last row has special values (each other reversed).
795          */
796         for (int i = 0; i < m; ++i) {
797             Hdata[i][0] = Hdata[i][0].subtract(hPowers[i]);
798             Hdata[m - 1][i] = Hdata[m - 1][i].subtract(hPowers[m - i - 1]);
799         }
800 
801         /*
802          * [1] states: "For 1/2 < h < 1 the bottom left element of the matrix should be (1 - 2*h^m +
803          * (2h - 1)^m )/m!" Since 0 <= h < 1, then if h > 1/2 is sufficient to check:
804          */
805         if (h.compareTo(BigFraction.ONE_HALF) == 1) {
806             Hdata[m - 1][0] = Hdata[m - 1][0].add(h.multiply(2).subtract(1).pow(m));
807         }
808 
809         /*
810          * Aside from the first column and last row, the (i, j)-th element is 1/(i - j + 1)! if i -
811          * j + 1 >= 0, else 0. 1's and 0's are already put, so only division with (i - j + 1)! is
812          * needed in the elements that have 1's. There is no need to calculate (i - j + 1)! and then
813          * divide - small steps avoid overflows. Note that i - j + 1 > 0 <=> i + 1 > j instead of
814          * j'ing all the way to m. Also note that it is started at g = 2 because dividing by 1 isn't
815          * really necessary.
816          */
817         for (int i = 0; i < m; ++i) {
818             for (int j = 0; j < i + 1; ++j) {
819                 if (i - j + 1 > 0) {
820                     for (int g = 2; g <= i - j + 1; ++g) {
821                         Hdata[i][j] = Hdata[i][j].divide(g);
822                     }
823                 }
824             }
825         }
826         return new Array2DRowFieldMatrix<BigFraction>(BigFractionField.getInstance(), Hdata);
827     }

829     /***
830      * Creates {@code H} of size {@code m x m} as described in [1] (see above)
831      * using double-precision.
832      *
833      * @param d statistic
834      * @param n sample size
835      * @return H matrix
836      * @throws NumberIsTooLargeException if fractional part is greater than 1
837      */
838     private RealMatrix createRoundedH(double d, int n)
839         throws NumberIsTooLargeException {
840 
841         final int k = (int) Math.ceil(n * d);
842         final int m = 2 * k - 1;
843         final double h = k - n * d;
844         if (h >= 1) {
845             throw new NumberIsTooLargeException(h, 1.0, false);
846         }
847         final double[][] Hdata = new double[m][m];
848 
849         /*
850          * Start by filling everything with either 0 or 1.
851          */
852         for (int i = 0; i < m; ++i) {
853             for (int j = 0; j < m; ++j) {
854                 if (i - j + 1 < 0) {
855                     Hdata[i][j] = 0;
856                 } else {
857                     Hdata[i][j] = 1;
858                 }
859             }
860         }
861 
862         /*
863          * Setting up power-array to avoid calculating the same value twice: hPowers[0] = h^1 ...
864          * hPowers[m-1] = h^m
865          */
866         final double[] hPowers = new double[m];
867         hPowers[0] = h;
868         for (int i = 1; i < m; ++i) {
869             hPowers[i] = h * hPowers[i - 1];
870         }
871 
872         /*
873          * First column and last row has special values (each other reversed).
874          */
875         for (int i = 0; i < m; ++i) {
876             Hdata[i][0] = Hdata[i][0] - hPowers[i];
877             Hdata[m - 1][i] -= hPowers[m - i - 1];
878         }
879 
880         /*
881          * [1] states: "For 1/2 < h < 1 the bottom left element of the matrix should be (1 - 2*h^m +
882          * (2h - 1)^m )/m!" Since 0 <= h < 1, then if h > 1/2 is sufficient to check:
883          */
884         if (Double.compare(h, 0.5) > 0) {
885             Hdata[m - 1][0] += FastMath.pow(2 * h - 1, m);
886         }
887 
888         /*
889          * Aside from the first column and last row, the (i, j)-th element is 1/(i - j + 1)! if i -
890          * j + 1 >= 0, else 0. 1's and 0's are already put, so only division with (i - j + 1)! is
891          * needed in the elements that have 1's. There is no need to calculate (i - j + 1)! and then
892          * divide - small steps avoid overflows. Note that i - j + 1 > 0 <=> i + 1 > j instead of
893          * j'ing all the way to m. Also note that it is started at g = 2 because dividing by 1 isn't
894          * really necessary.
895          */
896         for (int i = 0; i < m; ++i) {
897             for (int j = 0; j < i + 1; ++j) {
898                 if (i - j + 1 > 0) {
899                     for (int g = 2; g <= i - j + 1; ++g) {
900                         Hdata[i][j] /= g;
901                     }
902                 }
903             }
904         }
905         return MatrixUtils.createRealMatrix(Hdata);
906     }

925     /**
926      * Computes \( 1 + 2 \sum_{i=1}^\infty (-1)^i e^{-2 i^2 t^2} \) stopping when successive partial
927      * sums are within {@code tolerance} of one another, or when {@code maxIterations} partial sums
928      * have been computed. If the sum does not converge before {@code maxIterations} iterations a
929      * {@link TooManyIterationsException} is thrown.
930      *
931      * @param t argument
932      * @param tolerance Cauchy criterion for partial sums
933      * @param maxIterations maximum number of partial sums to compute
934      * @return Kolmogorov sum evaluated at t
935      * @throws TooManyIterationsException if the series does not converge
936      */
937     public double ksSum(double t, double tolerance, int maxIterations) {
938         if (t == 0.0) {
939             return 0.0;
940         }
941 
942         // TODO: for small t (say less than 1), the alternative expansion in part 3 of [1]
943         // from class javadoc should be used.
944 
945         final double x = -2 * t * t;
946         int sign = -1;
947         long i = 1;
948         double partialSum = 0.5d;
949         double delta = 1;
950         while (delta > tolerance && i < maxIterations) {
951             delta = FastMath.exp(x * i * i);
952             partialSum += sign * delta;
953             sign *= -1;
954             i++;
955         }
956         if (i == maxIterations) {
957             throw new TooManyIterationsException(maxIterations);
958         }
959         return partialSum * 2;
960     }

963      * Given a d-statistic in the range [0, 1] and the two sample sizes n and m,
964      * an integral d-statistic in the range [0, n*m] is calculated, that can be used for
965      * comparison with other integral d-statistics. Depending whether {@code strict} is
966      * {@code true} or not, the returned value divided by (n*m) is greater than
967      * (resp greater than or equal to) the given d value (allowing some tolerance).
968      *

970      * @param n first sample size
971      * @param m second sample size
972      * @param strict whether the returned value divided by (n*m) is allowed to be equal to d
973      * @return the integral d-statistic in the range [0, n*m]
974      */
975     private static long calculateIntegralD(double d, int n, int m, boolean strict) {
976         final double tol = 1e-12;  // d-values within tol of one another are considered equal
977         long nm = n * (long)m;

979         long lowerBound = (long)FastMath.floor((d + tol) * nm);
980         if (strict && lowerBound == upperBound) {
981             return upperBound + 1l;
982         }
983         else {
984             return upperBound;
985         }

1034     /**
1035      * Fills a boolean array randomly with a fixed number of {@code true} values.
1036      * The method uses a simplified version of the Fisher-Yates shuffle algorithm.
1037      * By processing first the {@code true} values followed by the remaining {@code false} values
1038      * less random numbers need to be generated. The method is optimized for the case
1039      * that the number of {@code true} values is larger than or equal to the number of
1040      * {@code false} values.
1041      *
1042      * @param b boolean array
1043      * @param numberOfTrueValues number of {@code true} values the boolean array should finally have
1044      * @param rng random data generator
1045      */
1046     private static void fillBooleanArrayRandomlyWithFixedNumberTrueValues(final boolean[] b, final int numberOfTrueValues, final UniformRandomProvider rng) {
1047         Arrays.fill(b, true);
1048         for (int k = numberOfTrueValues; k < b.length; k++) {
1049             final int r = rng.nextInt(k + 1);
1050             b[(b[r]) ? r : k] = false;
1051         }
1052     }

1055      * Uses Monte Carlo simulation to approximate \(P(D_{n,m} > d)\) where \(D_{n,m}\) is the
1056      * 2-sample Kolmogorov-Smirnov statistic. See
1057      * {@link #kolmogorovSmirnovStatistic(double[], double[])} for the definition of \(D_{n,m}\).
1058      * <p>
1059      * The simulation generates {@code iterations} random partitions of {@code m + n} into an
1060      * {@code n} set and an {@code m} set, computing \(D_{n,m}\) for each partition and returning
1061      * the proportion of values that are greater than {@code d}, or greater than or equal to
1062      * {@code d} if {@code strict} is {@code false}.
1063      * </p>
1064      *
1065      * @param d D-statistic value
1066      * @param n first sample size

1069      * @param strict whether or not the probability to compute is expressed as a strict inequality
1070      * @return proportion of randomly generated m-n partitions of m + n that result in \(D_{n,m}\)
1071      *         greater than (resp. greater than or equal to) {@code d}
1072      */

1074                               final int iterations) {

1078     /**
1079      * Uses Monte Carlo simulation to approximate \(P(D_{n,m} >= d/(n*m))\) where \(D_{n,m}\) is the
1080      * 2-sample Kolmogorov-Smirnov statistic.
1081      * <p>
1082      * Here d is the D-statistic represented as long value.
1083      * The real D-statistic is obtained by dividing d by n*m.
1084      * See also {@link #monteCarloP(double, int, int, boolean, int)}.
1085      *
1086      * @param d integral D-statistic
1087      * @param n first sample size
1088      * @param m second sample size
1089      * @param iterations number of random partitions to generate
1090      * @return proportion of randomly generated m-n partitions of m + n that result in \(D_{n,m}\)
1091      *         greater than or equal to {@code d/(n*m))}
1092      */
1093     private double integralMonteCarloP(final long d, final int n, final int m, final int iterations) {
1094 
1095         // ensure that nn is always the max of (n, m) to require fewer random numbers
1096         final int nn = FastMath.max(n, m);
1097         final int mm = FastMath.min(n, m);
1098         final int sum = nn + mm;
1099 
1100         int tail = 0;
1101         final boolean b[] = new boolean[sum];
1102         for (int i = 0; i < iterations; i++) {
1103             fillBooleanArrayRandomlyWithFixedNumberTrueValues(b, nn, rng);
1104             long curD = 0l;
1105             for(int j = 0; j < b.length; ++j) {
1106                 if (b[j]) {
1107                     curD += mm;
1108                     if (curD >= d) {
1109                         tail++;
1110                         break;
1111                     }
1112                 } else {
1113                     curD -= nn;
1114                     if (curD <= -d) {
1115                         tail++;
1116                         break;
1117                     }
1118                 }
1119             }
1120         }
1121         return (double) tail / iterations;
1122     }

1124     /**
1125      * If there are no ties in the combined dataset formed from x and y, this
1126      * method is a no-op.  If there are ties, a uniform random deviate in
1127      * (-minDelta / 2, minDelta / 2) - {0} is added to each value in x and y, where
1128      * minDelta is the minimum difference between unequal values in the combined
1129      * sample.  A fixed seed is used to generate the jitter, so repeated activations
1130      * with the same input arrays result in the same values.
1131      *
1132      * NOTE: if there are ties in the data, this method overwrites the data in
1133      * x and y with the jittered values.
1134      *
1135      * @param x first sample
1136      * @param y second sample
1137      */
1138     private static void fixTies(double[] x, double[] y) {
1139        final double[] values = MathArrays.unique(MathArrays.concatenate(x,y));
1140        if (values.length == x.length + y.length) {
1141            return;  // There are no ties
1142        }
1143 
1144        // Find the smallest difference between values, or 1 if all values are the same
1145        double minDelta = 1;
1146        double prev = values[0];
1147        double delta = 1;
1148        for (int i = 1; i < values.length; i++) {
1149           delta = prev - values[i];
1150           if (delta < minDelta) {
1151               minDelta = delta;
1152           }
1153           prev = values[i];
1154        }
1155        minDelta /= 2;
1156 
1157        // Add jitter using a fixed seed (so same arguments always give same results),
1158        // low-initialization-overhead generator
1159        final RealDistribution.Sampler sampler =
1160            new UniformRealDistribution(-minDelta, minDelta).createSampler(RandomSource.create(RandomSource.JDK, 100));
1161 
1162        // It is theoretically possible that jitter does not break ties, so repeat
1163        // until all ties are gone.  Bound the loop and throw MIE if bound is exceeded.
1164        int ct = 0;
1165        boolean ties = true;
1166        do {
1167            jitter(x, sampler);
1168            jitter(y, sampler);
1169            ties = hasTies(x, y);
1170            ct++;
1171        } while (ties && ct < 1000);
1172        if (ties) {
1173            throw new MathInternalError(); // Should never happen
1174        }
1175     }

1217     /**
1218      * The function C(i, j) defined in [4] (class javadoc), formula (5.5).
1219      * defined to return 1 if |i/n - j/m| <= c; 0 otherwise. Here c is scaled up
1220      * and recoded as a long to avoid rounding errors in comparison tests, so what
1221      * is actually tested is |im - jn| <= cmn.
1222      *
1223      * @param i first path parameter
1224      * @param j second path paramter
1225      * @param m first sample size
1226      * @param n second sample size
1227      * @param cmn integral D-statistic (see {@link #calculateIntegralD(double, int, int, boolean)})
1228      * @param strict whether or not the null hypothesis uses strict inequality
1229      * @return C(i,j) for given m, n, c
1230      */
1231     private static int c(int i, int j, int m, int n, long cmn, boolean strict) {
1232         if (strict) {
1233             return FastMath.abs(i*(long)n - j*(long)m) <= cmn ? 1 : 0;
1234         }
1235         return FastMath.abs(i*(long)n - j*(long)m) < cmn ? 1 : 0;
1236     }

1238     /**
1239      * The function N(i, j) defined in [4] (class javadoc).
1240      * Returns the number of paths over the lattice {(i,j) : 0 <= i <= n, 0 <= j <= m}
1241      * from (0,0) to (i,j) satisfying C(h,k, m, n, c) = 1 for each (h,k) on the path.
1242      * The return value is integral, but subject to overflow, so it is maintained and
1243      * returned as a double.
1244      *
1245      * @param i first path parameter
1246      * @param j second path parameter
1247      * @param m first sample size
1248      * @param n second sample size
1249      * @param cnm integral D-statistic (see {@link #calculateIntegralD(double, int, int, boolean)})
1250      * @param strict whether or not the null hypothesis uses strict inequality
1251      * @return number or paths to (i, j) from (0,0) representing D-values as large as c for given m, n
1252      */
1253     private static double n(int i, int j, int m, int n, long cnm, boolean strict) {
1254         /*
1255          * Unwind the recursive definition given in [4].
1256          * Compute n(1,1), n(1,2)...n(2,1), n(2,2)... up to n(i,j), one row at a time.
1257          * When n(i,*) are being computed, lag[] holds the values of n(i - 1, *).
1258          */
1259         final double[] lag = new double[n];
1260         double last = 0;
1261         for (int k = 0; k < n; k++) {
1262             lag[k] = c(0, k + 1, m, n, cnm, strict);
1263         }
1264         for (int k = 1; k <= i; k++) {
1265             last = c(k, 0, m, n, cnm, strict);
1266             for (int l = 1; l <= j; l++) {
1267                 lag[l - 1] = c(k, l, m, n, cnm, strict) * (last + lag[l - 1]);
1268                 last = lag[l - 1];
1269             }
1270         }
1271         return last;
1272     }
