37  /**
38   * Provides percentile computation.
39   * <p>
40   * There are several commonly used methods for estimating percentiles (a.k.a.
41   * quantiles) based on sample data.  For large samples, the different methods
42   * agree closely, but when sample sizes are small, different methods will give
43   * significantly different results.  The algorithm implemented here works as follows:
44   * <ol>
45   * <li>Let <code>n</code> be the length of the (sorted) array and
46   * <code>0 < p <= 100</code> be the desired percentile.</li>
47   * <li>If <code> n = 1 </code> return the unique array element (regardless of
48   * the value of <code>p</code>); otherwise </li>
49   * <li>Compute the estimated percentile position
50   * <code> pos = p * (n + 1) / 100</code> and the difference, <code>d</code>
51   * between <code>pos</code> and <code>floor(pos)</code> (i.e. the fractional
52   * part of <code>pos</code>).</li>
53   * <li> If <code>pos < 1</code> return the smallest element in the array.</li>
54   * <li> Else if <code>pos >= n</code> return the largest element in the array.</li>
55   * <li> Else let <code>lower</code> be the element in position
56   * <code>floor(pos)</code> in the array and let <code>upper</code> be the
57   * next element in the array.  Return <code>lower + d * (upper - lower)</code>
58   * </li>
59   * </ol></p>
60   * <p>
61   * To compute percentiles, the data must be at least partially ordered.  Input
62   * arrays are copied and recursively partitioned using an ordering definition.
63   * The ordering used by <code>Arrays.sort(double[])</code> is the one determined
64   * by {@link java.lang.Double#compareTo(Double)}.  This ordering makes
65   * <code>Double.NaN</code> larger than any other value (including
66   * <code>Double.POSITIVE_INFINITY</code>).  Therefore, for example, the median
67   * (50th percentile) of
68   * <code>{0, 1, 2, 3, 4, Double.NaN}</code> evaluates to <code>2.5.</code></p>
69   * <p>
70   * Since percentile estimation usually involves interpolation between array
71   * elements, arrays containing  <code>NaN</code> or infinite values will often
72   * result in <code>NaN</code> or infinite values returned.</p>
73   * <p>
74   * Further, to include different estimation types such as R1, R2 as mentioned in
75   * <a href="http://en.wikipedia.org/wiki/Quantile">Quantile page(wikipedia)</a>,
76   * a type specific NaN handling strategy is used to closely match with the
77   * typically observed results from popular tools like R(R1-R9), Excel(R7).</p>
78   * <p>
79   * Since 2.2, Percentile uses only selection instead of complete sorting
80   * and caches selection algorithm state between calls to the various
81   * {@code evaluate} methods. This greatly improves efficiency, both for a single
82   * percentile and multiple percentile computations. To maximize performance when
83   * multiple percentiles are computed based on the same data, users should set the
84   * data array once using either one of the {@link #evaluate(double[], double)} or
85   * {@link #setData(double[])} methods and thereafter {@link #evaluate(double)}
86   * with just the percentile provided.
87   * </p>
88   * <p>
89   * <strong>Note that this implementation is not synchronized.</strong> If
90   * multiple threads access an instance of this class concurrently, and at least
91   * one of the threads invokes the <code>increment()</code> or
92   * <code>clear()</code> method, it must be synchronized externally.</p>
93   */

305      /**
306      * Returns an estimate of the <code>p</code>th percentile of the values
307      * in the <code>values</code> array, starting with the element in (0-based)
308      * position <code>begin</code> in the array and including <code>length</code>
309      * values.
310      * <p>
311      * Calls to this method do not modify the internal <code>quantile</code>
312      * state of this statistic.</p>
313      * <p>
314      * <ul>
315      * <li>Returns <code>Double.NaN</code> if <code>length = 0</code></li>
316      * <li>Returns (for any value of <code>p</code>) <code>values[begin]</code>
317      *  if <code>length = 1 </code></li>
318      * <li>Throws <code>MathIllegalArgumentException</code> if <code>values</code>
319      *  is null , <code>begin</code> or <code>length</code> is invalid, or
320      * <code>p</code> is not a valid quantile value (p must be greater than 0
321      * and less than or equal to 100)</li>
322      * </ul></p>
323      * <p>
324      * See {@link Percentile} for a description of the percentile estimation
325      * algorithm used.</p>
326      *
327      * @param values array of input values
328      * @param p  the percentile to compute
329      * @param begin  the first (0-based) element to include in the computation
330      * @param length  the number of array elements to include
331      * @return  the percentile value
332      * @throws MathIllegalArgumentException if the parameters are not valid or the
333      * input array is null
334      */
335     public double evaluate(final double[] values, final int begin,
336                            final int length, final double p)
337         throws MathIllegalArgumentException {
338 
339         MathArrays.verifyValues(values, begin, length);
340         if (p > 100 || p <= 0) {
341             throw new OutOfRangeException(LocalizedFormats.OUT_OF_BOUNDS_QUANTILE_VALUE, p, 0, 100);
342         }
343         if (length == 0) {
344             return Double.NaN;
345         }
346         if (length == 1) {
347             return values[begin]; // always return single value for n = 1
348         }
349 
350         final double[] work = getWorkArray(values, begin, length);
351         final int[] pivotsHeap = getPivots(values);
352         return work.length == 0 ? Double.NaN :
353                     estimationType.evaluate(work, pivotsHeap, p, kthSelector);
354     }

390     /**
391      * Get the work array to operate. Makes use of prior {@code storedData} if
392      * it exists or else do a check on NaNs and copy a subset of the array
393      * defined by begin and length parameters. The set {@link #nanStrategy} will
394      * be used to either retain/remove/replace any NaNs present before returning
395      * the resultant array.
396      *
397      * @param values the array of numbers
398      * @param begin index to start reading the array
399      * @param length the length of array to be read from the begin index
400      * @return work array sliced from values in the range [begin,begin+length)
401      * @throws MathIllegalArgumentException if values or indices are invalid
402      */
403     protected double[] getWorkArray(final double[] values, final int begin, final int length) {
404         final double[] work;
405         if (values == getDataRef()) {
406             work = getDataRef();
407         } else {
408             switch (nanStrategy) {
409                 case MAXIMAL:// Replace NaNs with +INFs
410                     work = replaceAndSlice(values, begin, length, Double.NaN, Double.POSITIVE_INFINITY);
411                     break;
412                 case MINIMAL:// Replace NaNs with -INFs
413                     work = replaceAndSlice(values, begin, length, Double.NaN, Double.NEGATIVE_INFINITY);
414                     break;
415                 case REMOVED:// Drop NaNs from data
416                     work = removeAndSlice(values, begin, length, Double.NaN);
417                     break;
418                 case FAILED:// just throw exception as NaN is un-acceptable
419                     work = copyOf(values, begin, length);
420                     MathArrays.checkNotNaN(work);
421                     break;
422                 default: //FIXED
423                     work = copyOf(values,begin,length);
424                     break;
425             }
426         }
427         return work;
428     }

430     /**
431      * Make a copy of the array for the slice defined by array part from
432      * [begin, begin+length)
433      * @param values the input array
434      * @param begin start index of the array to include
435      * @param length number of elements to include from begin
436      * @return copy of a slice of the original array
437      */
438     private static double[] copyOf(final double[] values, final int begin, final int length) {
439         MathArrays.verifyValues(values, begin, length);
440         return MathArrays.copyOfRange(values, begin, begin + length);
441     }

443     /**
444      * Replace every occurrence of a given value with a replacement value in a
445      * copied slice of array defined by array part from [begin, begin+length).
446      * @param values the input array
447      * @param begin start index of the array to include
448      * @param length number of elements to include from begin
449      * @param original the value to be replaced with
450      * @param replacement the value to be used for replacement
451      * @return the copy of sliced array with replaced values
452      */
453     private static double[] replaceAndSlice(final double[] values,
454                                             final int begin, final int length,
455                                             final double original,
456                                             final double replacement) {
457         final double[] temp = copyOf(values, begin, length);
458         for(int i = 0; i < length; i++) {
459             temp[i] = Precision.equalsIncludingNaN(original, temp[i]) ?
460                       replacement : temp[i];
461         }
462         return temp;
463     }

465     /**
466      * Remove the occurrence of a given value in a copied slice of array
467      * defined by the array part from [begin, begin+length).
468      * @param values the input array
469      * @param begin start index of the array to include
470      * @param length number of elements to include from begin
471      * @param removedValue the value to be removed from the sliced array
472      * @return the copy of the sliced array after removing the removedValue
473      */
474     private static double[] removeAndSlice(final double[] values,
475                                            final int begin, final int length,
476                                            final double removedValue) {
477         MathArrays.verifyValues(values, begin, length);
478         final double[] temp;
479         //BitSet(length) to indicate where the removedValue is located
480         final BitSet bits = new BitSet(length);
481         for (int i = begin; i < begin+length; i++) {
482             if (Precision.equalsIncludingNaN(removedValue, values[i])) {
483                 bits.set(i - begin);
484             }
485         }
486         //Check if empty then create a new copy
487         if (bits.isEmpty()) {
488             temp = copyOf(values, begin, length); // Nothing removed, just copy
489         } else if(bits.cardinality() == length) {
490             temp = new double[0];                 // All removed, just empty
491         }else {                                   // Some removable, so new
492             temp = new double[length - bits.cardinality()];
493             int start = begin;  //start index from source array (i.e values)
494             int dest = 0;       //dest index in destination array(i.e temp)
495             int nextOne = -1;   //nextOne is the index of bit set of next one
496             int bitSetPtr = 0;  //bitSetPtr is start index pointer of bitset
497             while ((nextOne = bits.nextSetBit(bitSetPtr)) != -1) {
498                 final int lengthToCopy = nextOne - bitSetPtr;
499                 System.arraycopy(values, start, temp, dest, lengthToCopy);
500                 dest += lengthToCopy;
501                 start = begin + (bitSetPtr = bits.nextClearBit(nextOne));
502             }
503             //Copy any residue past start index till begin+length
504             if (start < begin + length) {
505                 System.arraycopy(values,start,temp,dest,begin + length - start);
506             }
507         }
508         return temp;
509     }

511     /**
512      * Get pivots which is either cached or a newly created one
513      *
514      * @param values array containing the input numbers
515      * @return cached pivots or a newly created one
516      */
517     private int[] getPivots(final double[] values) {
518         final int[] pivotsHeap;
519         if (values == getDataRef()) {
520             pivotsHeap = cachedPivots;
521         } else {
522             pivotsHeap = new int[PIVOTS_HEAP_LENGTH];
523             Arrays.fill(pivotsHeap, -1);
524         }
525         return pivotsHeap;
526     }

636     /**
637      * An enum for various estimation strategies of a percentile referred in
638      * <a href="http://en.wikipedia.org/wiki/Quantile">wikipedia on quantile</a>
639      * with the names of enum matching those of types mentioned in
640      * wikipedia.
641      * <p>
642      * Each enum corresponding to the specific type of estimation in wikipedia
643      * implements  the respective formulae that specializes in the below aspects
644      * <ul>
645      * <li>An <b>index method</b> to calculate approximate index of the
646      * estimate</li>
647      * <li>An <b>estimate method</b> to estimate a value found at the earlier
648      * computed index</li>
649      * <li>A <b> minLimit</b> on the quantile for which first element of sorted
650      * input is returned as an estimate </li>
651      * <li>A <b> maxLimit</b> on the quantile for which last element of sorted
652      * input is returned as an estimate </li>
653      * </ul>
654      * <p>
655      * Users can now create {@link Percentile} by explicitly passing this enum;
656      * such as by invoking {@link Percentile#withEstimationType(EstimationType)}
657      * <p>
658      * References:
659      * <ol>
660      * <li>
661      * <a href="http://en.wikipedia.org/wiki/Quantile">Wikipedia on quantile</a>
662      * </li>
663      * <li>
664      * <a href="https://www.amherst.edu/media/view/129116/.../Sample+Quantiles.pdf">
665      * Hyndman, R. J. and Fan, Y. (1996) Sample quantiles in statistical
666      * packages, American Statistician 50, 361–365</a> </li>
667      * <li>
668      * <a href="http://stat.ethz.ch/R-manual/R-devel/library/stats/html/quantile.html">
669      * R-Manual </a></li>
670      * </ol>
671      */
672     public enum EstimationType {
673         /**
674          * This is the default type used in the {@link Percentile}.This method
675          * has the following formulae for index and estimates<br>
676          * \( \begin{align}
677          * &amp;index    = (N+1)p\ \\
678          * &amp;estimate = x_{\lceil h\,-\,1/2 \rceil} \\
679          * &amp;minLimit = 0 \\
680          * &amp;maxLimit = 1 \\
681          * \end{align}\)
682          */
683         LEGACY("Legacy Apache Commons Math") {
684             /**
685              * {@inheritDoc}.This method in particular makes use of existing
686              * Apache Commons Math style of picking up the index.
687              */
688             @Override
689             protected double index(final double p, final int length) {
690                 final double minLimit = 0d;
691                 final double maxLimit = 1d;
692                 return Double.compare(p, minLimit) == 0 ? 0 :
693                        Double.compare(p, maxLimit) == 0 ?
694                                length : p * (length + 1);
695             }
696         },
697         /**
698          * The method R_1 has the following formulae for index and estimates<br>
699          * \( \begin{align}
700          * &amp;index= Np + 1/2\,  \\
701          * &amp;estimate= x_{\lceil h\,-\,1/2 \rceil} \\
702          * &amp;minLimit = 0 \\
703          * \end{align}\)
704          */
705         R_1("R-1") {
706 
707             @Override
708             protected double index(final double p, final int length) {
709                 final double minLimit = 0d;
710                 return Double.compare(p, minLimit) == 0 ? 0 : length * p + 0.5;
711             }
712 
713             /**
714              * {@inheritDoc}This method in particular for R_1 uses ceil(pos-0.5)
715              */
716             @Override
717             protected double estimate(final double[] values,
718                                       final int[] pivotsHeap, final double pos,
719                                       final int length, final KthSelector selector) {
720                 return super.estimate(values, pivotsHeap, FastMath.ceil(pos - 0.5), length, selector);
721             }
722 
723         },
724         /**
725          * The method R_2 has the following formulae for index and estimates<br>
726          * \( \begin{align}
727          * &amp;index= Np + 1/2\, \\
728          * &amp;estimate=\frac{x_{\lceil h\,-\,1/2 \rceil} +
729          * x_{\lfloor h\,+\,1/2 \rfloor}}{2} \\
730          * &amp;minLimit = 0 \\
731          * &amp;maxLimit = 1 \\
732          * \end{align}\)
733          */
734         R_2("R-2") {
735 
736             @Override
737             protected double index(final double p, final int length) {
738                 final double minLimit = 0d;
739                 final double maxLimit = 1d;
740                 return Double.compare(p, maxLimit) == 0 ? length :
741                        Double.compare(p, minLimit) == 0 ? 0 : length * p + 0.5;
742             }
743 
744             /**
745              * {@inheritDoc}This method in particular for R_2 averages the
746              * values at ceil(p+0.5) and floor(p-0.5).
747              */
748             @Override
749             protected double estimate(final double[] values,
750                                       final int[] pivotsHeap, final double pos,
751                                       final int length, final KthSelector selector) {
752                 final double low =
753                         super.estimate(values, pivotsHeap, FastMath.ceil(pos - 0.5), length, selector);
754                 final double high =
755                         super.estimate(values, pivotsHeap,FastMath.floor(pos + 0.5), length, selector);
756                 return (low + high) / 2;
757             }
758 
759         },
760         /**
761          * The method R_3 has the following formulae for index and estimates<br>
762          * \( \begin{align}
763          * &amp;index= Np \\
764          * &amp;estimate= x_{\lfloor h \rceil}\, \\
765          * &amp;minLimit = 0.5/N \\
766          * \end{align}\)
767          */
768         R_3("R-3") {
769             @Override
770             protected double index(final double p, final int length) {
771                 final double minLimit = 1d/2 / length;
772                 return Double.compare(p, minLimit) <= 0 ?
773                         0 : FastMath.rint(length * p);
774             }
775 
776         },
777         /**
778          * The method R_4 has the following formulae for index and estimates<br>
779          * \( \begin{align}
780          * &amp;index= Np\, \\
781          * &amp;estimate= x_{\lfloor h \rfloor} + (h -
782          * \lfloor h \rfloor) (x_{\lfloor h \rfloor + 1} - x_{\lfloor h
783          * \rfloor}) \\
784          * &amp;minLimit = 1/N \\
785          * &amp;maxLimit = 1 \\
786          * \end{align}\)
787          */
788         R_4("R-4") {
789             @Override
790             protected double index(final double p, final int length) {
791                 final double minLimit = 1d / length;
792                 final double maxLimit = 1d;
793                 return Double.compare(p, minLimit) < 0 ? 0 :
794                        Double.compare(p, maxLimit) == 0 ? length : length * p;
795             }
796 
797         },
798         /**
799          * The method R_5 has the following formulae for index and estimates<br>
800          * \( \begin{align}
801          * &amp;index= Np + 1/2\\
802          * &amp;estimate= x_{\lfloor h \rfloor} + (h -
803          * \lfloor h \rfloor) (x_{\lfloor h \rfloor + 1} - x_{\lfloor h
804          * \rfloor}) \\
805          * &amp;minLimit = 0.5/N \\
806          * &amp;maxLimit = (N-0.5)/N
807          * \end{align}\)
808          */
809         R_5("R-5") {
810 
811             @Override
812             protected double index(final double p, final int length) {
813                 final double minLimit = 1d/2 / length;
814                 final double maxLimit = (length - 0.5) / length;
815                 return Double.compare(p, minLimit) < 0 ? 0 :
816                        Double.compare(p, maxLimit) >= 0 ?
817                                length : length * p + 0.5;
818             }
819         },
820         /**
821          * The method R_6 has the following formulae for index and estimates<br>
822          * \( \begin{align}
823          * &amp;index= (N + 1)p \\
824          * &amp;estimate= x_{\lfloor h \rfloor} + (h -
825          * \lfloor h \rfloor) (x_{\lfloor h \rfloor + 1} - x_{\lfloor h
826          * \rfloor}) \\
827          * &amp;minLimit = 1/(N+1) \\
828          * &amp;maxLimit = N/(N+1) \\
829          * \end{align}\)
830          * <p>
831          * <b>Note:</b> This method computes the index in a manner very close to
832          * the default Commons Math Percentile existing implementation. However
833          * the difference to be noted is in picking up the limits with which
834          * first element (p&lt;1(N+1)) and last elements (p&gt;N/(N+1))are done.
835          * While in default case; these are done with p=0 and p=1 respectively.
836          */
837         R_6("R-6") {
838 
839             @Override
840             protected double index(final double p, final int length) {
841                 final double minLimit = 1d / (length + 1);
842                 final double maxLimit = 1d * length / (length + 1);
843                 return Double.compare(p, minLimit) < 0 ? 0 :
844                        Double.compare(p, maxLimit) >= 0 ?
845                                length : (length + 1) * p;
846             }
847         },
848 
849         /**
850          * The method R_7 implements Microsoft Excel style computation has the
851          * following formulae for index and estimates.<br>
852          * \( \begin{align}
853          * &amp;index = (N-1)p + 1 \\
854          * &amp;estimate = x_{\lfloor h \rfloor} + (h -
855          * \lfloor h \rfloor) (x_{\lfloor h \rfloor + 1} - x_{\lfloor h
856          * \rfloor}) \\
857          * &amp;minLimit = 0 \\
858          * &amp;maxLimit = 1 \\
859          * \end{align}\)
860          */
861         R_7("R-7") {
862             @Override
863             protected double index(final double p, final int length) {
864                 final double minLimit = 0d;
865                 final double maxLimit = 1d;
866                 return Double.compare(p, minLimit) == 0 ? 0 :
867                        Double.compare(p, maxLimit) == 0 ?
868                                length : 1 + (length - 1) * p;
869             }
870 
871         },
872 
873         /**
874          * The method R_8 has the following formulae for index and estimates<br>
875          * \( \begin{align}
876          * &amp;index = (N + 1/3)p + 1/3  \\
877          * &amp;estimate = x_{\lfloor h \rfloor} + (h -
878            \lfloor h \rfloor) (x_{\lfloor h \rfloor + 1} - x_{\lfloor h
879          * \rfloor}) \\
880          * &amp;minLimit = (2/3)/(N+1/3) \\
881          * &amp;maxLimit = (N-1/3)/(N+1/3) \\
882          * \end{align}\)
883          * <p>
884          * As per Ref [2,3] this approach is most recommended as it provides
885          * an approximate median-unbiased estimate regardless of distribution.
886          */
887         R_8("R-8") {
888             @Override
889             protected double index(final double p, final int length) {
890                 final double minLimit = 2 * (1d / 3) / (length + 1d / 3);
891                 final double maxLimit =
892                         (length - 1d / 3) / (length + 1d / 3);
893                 return Double.compare(p, minLimit) < 0 ? 0 :
894                        Double.compare(p, maxLimit) >= 0 ? length :
895                            (length + 1d / 3) * p + 1d / 3;
896             }
897         },
898 
899         /**
900          * The method R_9 has the following formulae for index and estimates<br>
901          * \( \begin{align}
902          * &amp;index = (N + 1/4)p + 3/8\\
903          * &amp;estimate = x_{\lfloor h \rfloor} + (h -
904            \lfloor h \rfloor) (x_{\lfloor h \rfloor + 1} - x_{\lfloor h
905          * \rfloor}) \\
906          * &amp;minLimit = (5/8)/(N+1/4) \\
907          * &amp;maxLimit = (N-3/8)/(N+1/4) \\
908          * \end{align}\)
909          */
910         R_9("R-9") {
911             @Override
912             protected double index(final double p, final int length) {
913                 final double minLimit = 5d/8 / (length + 0.25);
914                 final double maxLimit = (length - 3d/8) / (length + 0.25);
915                 return Double.compare(p, minLimit) < 0 ? 0 :
916                        Double.compare(p, maxLimit) >= 0 ? length :
917                                (length + 0.25) * p + 3d/8;
918             }
919 
920         },

936          * Finds the index of array that can be used as starting index to
937          * {@link #estimate(double[], int[], double, int, KthSelector) estimate}
938          * percentile. The calculation of index calculation is specific to each
939          * {@link EstimationType}.
940          *
941          * @param p the p<sup>th</sup> quantile
942          * @param length the total number of array elements in the work array
943          * @return a computed real valued index as explained in the wikipedia
944          */
945         protected abstract double index(final double p, final int length);

947         /**
948          * Estimation based on K<sup>th</sup> selection. This may be overridden
949          * in specific enums to compute slightly different estimations.
950          *
951          * @param work array of numbers to be used for finding the percentile
952          * @param pos indicated positional index prior computed from calling
953          *            {@link #index(double, int)}
954          * @param pivotsHeap an earlier populated cache if exists; will be used
955          * @param length size of array considered
956          * @param selector a {@link KthSelector} used for pivoting during search
957          * @return estimated percentile
958          */
959         protected double estimate(final double[] work, final int[] pivotsHeap,
960                                   final double pos, final int length,
961                                   final KthSelector selector) {
962 
963             final double fpos = FastMath.floor(pos);
964             final int intPos = (int) fpos;
965             final double dif = pos - fpos;
966 
967             if (pos < 1) {
968                 return selector.select(work, pivotsHeap, 0);
969             }
970             if (pos >= length) {
971                 return selector.select(work, pivotsHeap, length - 1);
972             }
973 
974             final double lower = selector.select(work, pivotsHeap, intPos - 1);
975             final double upper = selector.select(work, pivotsHeap, intPos);
976             return lower + dif * (upper - lower);
977         }

979         /**
980          * Evaluate method to compute the percentile for a given bounded array
981          * using earlier computed pivots heap.<br>
982          * This basically calls the {@link #index(double, int) index} and then
983          * {@link #estimate(double[], int[], double, int, KthSelector) estimate}
984          * functions to return the estimated percentile value.
985          *
986          * @param work array of numbers to be used for finding the percentile
987          * @param pivotsHeap a prior cached heap which can speed up estimation
988          * @param p the p<sup>th</sup> quantile to be computed
989          * @param selector a {@link KthSelector} used for pivoting during search
990          * @return estimated percentile
991          * @throws OutOfRangeException if p is out of range
992          * @throws NullArgumentException if work array is null
993          */
994         protected double evaluate(final double[] work, final int[] pivotsHeap, final double p,
995                                   final KthSelector selector) {
996             MathUtils.checkNotNull(work);
997             if (p > 100 || p <= 0) {
998                 throw new OutOfRangeException(LocalizedFormats.OUT_OF_BOUNDS_QUANTILE_VALUE,
999                                               p, 0, 100);
1000             }
1001             return estimate(work, pivotsHeap, index(p/100d, work.length), work.length, selector);
1002         }

1004         /**
1005          * Evaluate method to compute the percentile for a given bounded array.
1006          * This basically calls the {@link #index(double, int) index} and then
1007          * {@link #estimate(double[], int[], double, int, KthSelector) estimate}
1008          * functions to return the estimated percentile value. Please
1009          * note that this method does not make use of cached pivots.
1010          *
1011          * @param work array of numbers to be used for finding the percentile
1012          * @param p the p<sup>th</sup> quantile to be computed
1013          * @return estimated percentile
1014          * @param selector a {@link KthSelector} used for pivoting during search
1015          * @throws OutOfRangeException if length or p is out of range
1016          * @throws NullArgumentException if work array is null
1017          */
1018         public double evaluate(final double[] work, final double p, final KthSelector selector) {
1019             return this.evaluate(work, null, p, selector);
